< 전체 구조 > 
브라우저 / 프론트 요청
→ run.py
→ app/__init__.py
→ api/auth_routes.py 또는 api/member_routes.py
→ schemas/ 에서 입력값 검사
→ services/ 에서 실제 처리
→ repositories/ 에서 DB 접근
→ models/ 에서 테이블 구조 사용
→ common/response.py 로 응답 반환

< 구현되는 기능 목록 >

이 구조로 만들면 아래 기능이 구현된다.

1. 인증
1-1. 회원가입
1-2. 로그인
1-3. 내 정보 조회

2. 로그인 보안
2-1. 로그인 실패 횟수 증가
2-2. 5회 실패 시 1분 잠금

3. 회원관리
3-1. 회원 목록 조회
3-2. 회원 상세 조회
3-3. 회원 정보 수정
3-4. 회원 비활성화
3-5. 회원 탈퇴(소프트 삭제)
